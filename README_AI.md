# README_AI


## Rules for AI Agents

- Always ask before attempting to running a command
- Never add/comment git changes, ask if you want to do so
- Never add unecessary changes -- only add code that are necessary and relevant to the change requested. Never do code polishing/clean up unless being asked to do it.

Before editing code:
- Identify the relevant files.
- Explain the change briefly.
- Then propose edits.


## Project structure

This project is my homepage build with Jekyll.

- `_data/`: data for the project
- `_includes/`: includes for the project
- `_layouts/`: layouts for the project
- `assets/`: css style files, fonts, iamges, javascripts, do not modify files in `assets/images/`

- `pages/`: contents for each page (corresponding to each tab in `_data/navigation.yml`)
- `documents/`: my personal documents (presentation materials, reports) to be shared in this website
- `notes/`: my personal notes
- `research/`: my research project webpages

- `index.md`: homepage contents

- `.cursorignore`: files to ignore in cursor
- `.gitignore`: files to ignore in git
- `.ruby-version`: Ruby version

- `Gemfile`: gemfile for the project
- `Gemfile.lock`: gemfile.lock for the project
- `README.md`: README for the project
- `README_AI.md`: README for the AI agents
